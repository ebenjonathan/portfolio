I Cut My Load Time in Half… Here's How

Jonathan Mupini April 2026 5 min read

I ran Lighthouse on this portfolio site. LCP came back at 5.8 seconds. The threshold for a "Good" rating is under 2.5 seconds. This article is a step-by-step record of what I changed and why each change mattered.

LCP stands for Largest Contentful Paint. It measures when the main visible content appears on screen. A slow LCP feels like a slow page even if the rest loads quickly.

The Diagnosis

Lighthouse flagged three categories of problems:

1. Render-blocking resources

Google Fonts loaded with a standard <link rel="stylesheet">. The browser stops rendering and waits for the font file to download before it shows any text. This alone added over a second to the LCP.

2. Unused JavaScript

Four vendor libraries were included in <script> tags but were never used anywhere on the page:

  • GLightbox, a lightbox library (~59KB)
  • Isotope Layout, a grid filtering library (~55KB)
  • Swiper, a slider library (~135KB)
  • imagesLoaded, a utility (~7KB)

Combined: ~256KB of JavaScript the browser downloaded, parsed, and tried to execute for nothing. These were left over from a template that included every possible feature. None of the corresponding HTML classes or initialisers existed anywhere in the page.

3. No image optimisation

All images were PNG. No WebP. Below-fold images had no loading="lazy" attribute.

There were also smaller issues: all scripts loaded synchronously (no defer), no critical CSS was inlined, and the CSS files were unminified.

What I Fixed

Fix 1: Async font loading

I replaced the blocking rel="stylesheet" with a preload trick that is well-supported across all modern browsers:

<link rel="preload" as="style"
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700;800&display=swap"
  onload="this.onload=null;this.rel='stylesheet'">
<noscript>
  <link rel="stylesheet" href="https://fonts.googleapis.com/css2?...">
</noscript>

The browser preloads the font in the background. It does not block the render. The <noscript> fallback handles visitors with JavaScript disabled.

Fix 2: Remove unused vendor bundles

I searched the entire HTML body for any use of GLightbox, Isotope, Swiper, or imagesLoaded. No class names. No JavaScript calls. No data attributes. Nothing.

I removed four <script> tags and two <link> tags for their CSS files. This cut roughly 256KB from the JavaScript payload and removed two render-blocking CSS requests from the critical path.

Fix 3: Defer all scripts

I added defer to every remaining <script> tag:

<script defer src="assets/vendor/bootstrap/js/bootstrap.bundle.min.js"></script>
<script defer src="assets/vendor/aos/aos.js"></script>
<script defer src="assets/js/main.js"></script>

Deferred scripts download in parallel with HTML parsing. They execute after the HTML is fully parsed. The page renders without waiting for any script. This is a one-line change per script tag with almost no downside.

Fix 4: Inline critical CSS

The hero section and preloader need their styles before any external CSS loads. I moved those rules into a <style> block in <head>. This is roughly 1.2KB of CSS that prevents a flash of unstyled content on the first paint.

I did not inline everything, only what is visible above the fold before the external CSS loads.

Fix 5: WebP images

I converted the site logo from PNG to WebP using Python's Pillow library:

from PIL import Image
img = Image.open("mupinilabs-logo.png")
img.save("mupinilabs-logo.webp", "WEBP", quality=85, method=6)

The WebP file was 16% smaller than the PNG. I updated the footer to use a <picture> element with the WebP as the primary source and PNG as fallback. I also added loading="lazy" since the logo is below the fold:

<picture>
  <source srcset="assets/img/mupinilabs-logo.webp" type="image/webp">
  <img src="assets/img/mupinilabs-logo.png" alt="MupiniLabs"
       loading="lazy" width="120" height="40">
</picture>

Fix 6: Minify CSS and JS

I minified the main CSS and JavaScript files using Python tools (csscompressor and jsmin). The results:

  • main.css: −30% (32KB → 22KB)
  • custom.css: −23% (13.5KB → 10.4KB)
  • main.js: −33% (after removing dead code from unused libraries)

Summary of changes

  • Fonts: render-blocking → async preload with noscript fallback
  • Removed 4 unused vendor JS bundles and 2 unused vendor CSS files
  • Added defer to all 7 remaining script tags
  • 1.2KB of critical CSS inlined in <head>
  • Logo: PNG → WebP with <picture> fallback, lazy-loaded
  • main.css −30%, custom.css −23%, main.js −33%

What I Learned

Fix fonts first. Render-blocking Google Fonts is the most common cause of slow LCP on small sites. It is also the easiest fix. The preload trick takes five minutes to implement.

Audit your vendor includes. Every project I have reviewed has at least one library that is loaded but unused. These accumulate when features are removed but their dependencies are not. A 10-minute search can remove hundreds of kilobytes.

defer on script tags is a one-line win. It improves load performance on every script in the page. There is almost no reason not to do it. The only exception is scripts that must run before the DOM is ready, which is rare.

Inline only the critical CSS, not everything. You do not need to inline your entire stylesheet. Inline only the styles for what the user sees on first load: the hero section, the nav, and any preloader. The rest can load from an external file.
