Building a Centralized XDR System with Wazuh

Role: Lead Developer

Scope: End-to-end system design, backend, frontend, deployment

I designed and deployed an Extended Detection and Response (XDR) system to eliminate blind spots in security monitoring and enable real-time threat detection across endpoints.

The Problem

The environment had no centralized visibility. Logs were scattered across systems, making it almost impossible to detect suspicious behavior early or investigate incidents efficiently.

When something went wrong, response was slow and reactive. There was no unified way to correlate events or identify patterns across machines.

My Approach

Instead of patching individual issues, I focused on building a centralized security layer that could aggregate, analyze, and respond to events in real time.

I implemented a Wazuh-based XDR system that collects logs from multiple endpoints, applies detection rules, and surfaces actionable alerts through a single dashboard.

What I Built

  • Deployed a Wazuh manager for centralized log aggregation and analysis
  • Installed and configured agents across endpoints for continuous monitoring
  • Integrated Elastic Stack for indexing, search, and visualization
  • Created custom detection rules for authentication anomalies and system threats
  • Built dashboards for real-time monitoring and incident visibility

Key Decisions

  • Chose centralized logging over distributed scripts to improve visibility and control
  • Prioritized real-time alerting to reduce detection delays
  • Focused on actionable alerts to avoid noise and alert fatigue

The Impact

  • Eliminated fragmented logging by centralizing all security events
  • Reduced incident detection time from reactive investigation to near real-time awareness
  • Enabled faster root cause analysis through correlated event data
  • Shifted security posture from reactive to proactive monitoring

Tech Stack

Wazuh, Elastic Stack, Linux, Log Analysis, Security Monitoring
